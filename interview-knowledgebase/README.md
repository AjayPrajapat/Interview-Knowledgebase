# Interview Knowledgebase

A complete Markdown knowledgebase for 40 interview question types across:

- Fullstack Engineering
- Business Development
- Branding and Marketing
- Product and Strategy

## How to Use This Knowledgebase

Each category file contains:

- What that question type tests
- Typical interview questions
- Key answer points
- Common mistakes
- Advanced follow-up prompts
- Real-world application
- A standardized answer framework applied to that category
- Question-by-question guidance so you can answer consistently

## Universal Answering Frameworks

Use the right structure based on question type.

### 1. Fundamental / Concept / Definition Questions (5 Steps)

1. Definition
2. How it works
3. Simple example
4. Real-world use case
5. Summary line

### 2. Implementation / Coding / Design Questions (6 Steps)

1. Clarify requirements and constraints
2. High-level approach
3. Step-by-step solution
4. Edge cases and risks
5. Validation (tests, metrics, monitoring)
6. Summary and tradeoff note

### 3. Comparison / Tradeoff Questions (5 Steps)

1. Decision criteria
2. Option A strengths and limits
3. Option B strengths and limits
4. Recommended option for this context
5. Risk and reversal condition

### 4. Debugging / Troubleshooting / RCA Questions (6 Steps)

1. Reproduce and scope
2. Gather evidence (logs, metrics, traces)
3. Build hypotheses
4. Isolate and confirm root cause
5. Fix and rollback plan
6. Prevention and monitoring

### 5. Product / Scenario / Behavioral Questions (STAR+)

1. Situation
2. Task
3. Action
4. Result (with metrics)
5. Reflection (what changed afterward)

## Answer Length Guidance

- Junior: 20-45 seconds
- Mid-level: 45-90 seconds
- Senior/Lead: 90-150 seconds with tradeoffs and metrics

## Practice Workflow

1. Pick one category file.
2. Answer 5-10 questions aloud using the category framework.
3. Record and check for structure, clarity, metrics, and relevance.
4. Rewrite weak answers with better examples.
5. Repeat until responses are concise and repeatable.
