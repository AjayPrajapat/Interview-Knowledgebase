# Contributing Guidelines

## Contribution Rules
1. Use templates from `templates/` for all new content.
2. Include YAML frontmatter in every topic file.
3. Keep one core concept per topic.
4. Add links to related topics, ADRs, and diagrams.
5. Pass markdown lint and link checks before merge.

## Pull Request Expectations
- Clear problem statement
- Scope and impacted categories
- Evidence of validation (lint/check output)
- Reviewer assignment from relevant ownership area

## Review Policy
- New category structure changes require architect reviewer.
- ADR is required for taxonomy or template breaking changes.
