# Versioning Strategy

This repository uses Semantic Versioning for releases.

## Release Semantics
- MAJOR: breaking taxonomy/template changes
- MINOR: new categories, sections, or topic additions
- PATCH: corrections, links, formatting, clarifications

## Topic-Level Versioning
Each topic frontmatter includes a `version` field to track local evolution.

## Changelog
All release notes must be recorded in `CHANGELOG.md`.
